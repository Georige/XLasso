## 核心问题：为什么CV选择错误的参数？

### 验证集MSE最优 ≠ 稀疏选择最优

CV在验证集上优化的目标是最小化MSE：

```
CV目标: min mean_squared_error(y_val, y_pred)
```

**但这产生了矛盾的结果**：

1. **增大lambda_ridge (e.g., 10.0)**
   - Ridge惩罚极强 → beta_ridge ≈ 0
   - 权重 = 1/(|beta_ridge|+eps)^gamma ≈ 常数
   - 所有特征权重几乎相同
   - Lasso必须选择大量特征来拟合数据（因为没有特征区分度）

2. **增大gamma (e.g., 2.0)**
   - 权重计算中幂次更高
   - 噪声特征权重不会衰减到很小
   - Lasso无法有效排除噪声特征

3. **CV选择的"最优"组合**
   - lambda_ridge=10, gamma=2.0, alpha~常量
   - 验证集MSE最小
   - 但代价：选择了40+个特征（真信号只有20个）

### 理论解释

```
当lambda_ridge → ∞:
  β_ridge → 0
  weights → 1/(eps)^γ ≈ 常数
  X_adaptive → X（无缩放）
  Lasso只能靠增大alpha来稀疏
  但增大alpha → 更少特征 → 验证MSE增大
  → 矛盾！只好选更多特征来平衡
```

**Fixed使用lambda_ridge=1.0, gamma=0.5时**：
- Stage1 Ridge保留权重差异
- 权重缩放有效区分信号vs噪声
- Stage2 Lasso可以精准稀疏选择

## 性能对比

| 配置 | n_selected | FDR | F1 |
|------|------------|-----|-----|
| **Truth** | 20 | 0.0 | 1.0 |
| AFL Fixed | **20.4** | **0.178** | **0.886** |
| AFL CV (Orig) | 41.7 | 0.395 | 0.720 |
| AFL CV (Tuned) | 38.9 | 0.429 | 0.707 |

**结论**：
- 缩小搜索空间（Tuned）略微减少了n_selected（41.7→38.9）
- 但FDR反而略增（0.395→0.429）
- 核心问题不在搜索空间，在于CV验证集优化的根本矛盾

## 建议

1. **继续使用Fixed参数** - Stage1最优参数已被验证更优
2. **如果必须用CV**：
   - 使用嵌套CV（外层评估泛化，内层选参数）
   - 或使用稳定性选择（Stability Selection）
3. **理论分析**：
   - 为什么lambda_ridge=1.0, gamma=0.5是最优的
   - 可能与n=300, p=500的比例有关
