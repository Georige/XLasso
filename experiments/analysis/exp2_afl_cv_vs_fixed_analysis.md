# AFL CV vs AFL Fixed 性能差距分析报告

## 实验配置

| 参数 | AFL Fixed | AFL CV |
|------|-----------|--------|
| lambda_ridge | 1.0 (固定) | [0.1, 1.0, 10.0] |
| gamma | 0.5 (固定) | [0.3, 0.5, 1.0, 2.0] |
| cv | N/A | 5-fold |
| n_alpha | N/A | 50 |
| 数据 | Exp2 AR(1), rho=0.8, n=300, p=500, n_nonzero=20 |

## 核心结果对比

### 高SNR (SNR=10.0, sigma=0.1) 性能对比

| 指标 | AFL Fixed | AFL CV | 差距 |
|------|-----------|--------|------|
| F1 | 0.990 ± 0.013 | 0.772 ± 0.083 | **-0.218** |
| FDR | 0.019 ± 0.026 | 0.312 ± 0.079 | **+0.293** |
| n_selected | 20.4 ± 0.5 | 43.4 ± 18.7 | **+23.0** |

## 关键发现

### 1. CV选择了过多特征 (Over-Selection)

**Fixed** 在高SNR时稳定选择 ~20 个特征（接近真实 n_nonzero=20），而 **CV** 选择了 32-74 个特征。

```
Fixed repeat-wise n_selected: [21, 20, 20, 21, 20]  (方差: 0.3)
CV repeat-wise n_selected:    [32.6, 26.0, 48.4, 38.2, 73.8]  (方差: 348.7)
```

CV的n_selected方差是Fixed的**1162倍**，说明CV极不稳定。

### 2. CV在高SNR时FDR仍然高达31%

即使在最优条件下（SNR=10），CV的FDR仍为0.312，意味着**选中的特征中有31%是假阳性**。而Fixed的FDR仅为0.019。

### 3. F1方差距差巨大

```
Fixed F1方差: 0.00018
CV F1方差:    0.00686 (38倍差距)
```

CV的每次实验结果波动剧烈，重复性差。

## 根因分析

### 原因1: CV内部参数选择过拟合

CV在选择最优参数时，以验证集上的表现作为标准。但问题是：

- **搜索空间过大**: 3 (lambda_ridge) × 4 (gamma) × 50 (n_alpha) = 600 种参数组合
- **验证集噪声**: 每个CV fold只用60%的数据训练，40%验证
- **选择偏差**: 在600个选项中选择最优，必然会过拟合到验证集的特殊噪声结构

### 原因2: Stage1 Ridge估计不稳定导致权重偏差

AFL的两阶段结构：
1. Stage1: Ridge回归估计特征权重
2. Stage2: 使用Stage1权重作为自适应权重进行Lasso

CV在每个fold内部运行Stage1时：
- 数据量减少到 n*4/5 = 240
- Ridge估计的权重噪声更大
- 导致Stage2的权重选择偏差被放大

### 原因3: n_alpha=50 网格过密

CV使用50个alpha值进行路径搜索，每个alpha对应不同的稀疏度。在小样本CV下：
- alpha路径的前10-15个值通常已经包含最优解
- 后面的alpha值会引入过稀疏解
- CV可能在噪声结构复杂时选择过稀疏的解（导致TPR下降）

### 原因4: 参数空间的偏向性

CV选择的"最优参数"往往偏向于：
- 较小的 lambda_ridge（允许Ridge过拟合到当前数据）
- 较大的 gamma（降低自适应惩罚强度）
- 较大的 alpha（选择更多特征）

这些选择在特定fold上表现良好，但泛化能力差。

## 为什么Fixed没有这些问题

Fixed使用固定的 lambda_ridge=1.0, gamma=0.5：
- Stage1 Ridge使用全部数据估计，权重更稳定
- 没有"选择最优"的过拟合问题
- 参数是经过理论分析或先验知识确定的

## 建议改进方向

### 1. 减小CV搜索空间
```
当前: lambda_ridge_list=[0.1, 1.0, 10.0], gamma_list=[0.3, 0.5, 1.0, 2.0]
建议: lambda_ridge_list=[0.5, 1.0, 2.0], gamma_list=[0.3, 0.5, 0.7]
```

### 2. 减少n_alpha或使用对数尺度
```python
# 使用对数尺度alpha路径，减少过稀疏区域采样
alphas = np.logspace(-3, 0, 30)  # 30个对数均匀分布
```

### 3. 增加嵌套CV验证
考虑使用嵌套CV：外层评估泛化性能，内层选择参数，避免参数选择过拟合。

### 4. 使用稳定性选择 (Stability Selection)
在CV基础上[text](vscode-webview://0lcsea2n4spca6ifrasq214bsctb1nn5qj1b6cm6757fajpetu1m/clear/NLasso/XLasso/experiments/analysis/exp2_afl_cv_1se_report.md)，增加稳定性选择步骤：
- 多次子采样
- 选择在多次子采样中都被选中的特征
- 提高特征的可靠性

## 结论

AFL CV性能差的主要原因不是"CV方法本身有问题"，而是**CV的搜索空间和评估方式在当前数据规模(n=300, p=500)下造成了过拟合**。

Fixed参数 (lambda_ridge=1.0, gamma=0.5) 经过精心选择，在Stage1提供了稳定的权重估计，使得Stage2的稀疏选择更加准确。

CV的600个参数组合 × 5-fold验证 = 3000次评估，在n=300的数据上，每fold只有240个样本用于验证，这不足以可靠地选择最优参数。

**核心建议**: 降低CV搜索空间的复杂度，或采用更鲁棒的正则化路径选择方法。
