## 4. 参数敏感性分析

### 4.1 F1 与参数的相关系数

| 参数 | 与 F1 的 Pearson 相关系数 | 解释 |
|------|--------------------------|------|
| lambda_ | **r = +0.599** | 强正相关：lambda_ 越大，F1 越高 |
| gamma | **r = -0.642** | 强负相关：gamma 越小，F1 越高 |
| lambda_ridge | r ≈ 0 | 几乎无相关性 |

### 4.2 参数影响解读

#### lambda_（Lasso 正则化强度）

```
lambda_=0.01  →  F1≈0.12~0.18（选 200~320 个特征，过拟合）
lambda_=0.1   →  F1≈0.13~0.42（选 75~286 个特征）
lambda_=0.5   →  F1≈0.14~0.85（选 27~252 个特征）
lambda_=1.0   →  F1≈0.16~0.95（选 22~247 个特征，最优）
```

lambda_ 是最关键的超参数。值越大稀疏性越强，但过大可能导致欠拟合。

#### gamma（权重分化强度）

```
gamma=0.5  →  F1≈0.12~0.95（权重分化弱，对称惩罚，lambda_ 主导）
gamma=1.0  →  F1≈0.15~0.73（标准分化）
gamma=2.0  →  F1≈0.12~0.16（分化过强，权重极端化导致失效）
```

gamma=2.0 时权重极端化，弱信号权重过大导致几乎所有特征被压零困难。gamma=0.5 时权重分化弱，近似对称惩罚，效果主要由 lambda_ 决定。

#### lambda_ridge（第一阶段 Ridge 强度）

lambda_ridge 在 [1.0, 50.0] 范围内对 F1 影响极小（相关系数 ≈ 0），说明算法对第一阶段 Ridge 强度不敏感。

## 6. 收敛性警告

实验过程中出现大量 "Objective did not converge" 警告，主要集中在：

- lambda_ridge=1.0 且 lambda_ ∈ [0.01, 0.1] 的配置
- 所有 gamma=2.0 的配置

**建议**: 将 `max_iter` 从 2000 增加至 5000，或在 `n_alpha > 1` 时使用 warm start 策略加速收敛。

## 7. Stage2 调参建议

基于 Stage1 结果，建议在以下区间进行 Stage2 细粒度调参：

```yaml
best_structural_region:
  lambda_ridge: [1.0, 50.0]   # 范围广，建议取中间值 10.0
  lambda_: [0.5, 1.0]          # 细粒度搜索: [0.6, 0.7, 0.8, 0.9, 1.0]
  gamma: [0.5, 1.0]            # 细粒度搜索: [0.5, 0.6, 0.7, 0.8, 0.9, 1.0]
```

Stage2 应使用交叉验证法（cv_folds=5）对 lambda_ 进行细粒度搜索，确定最终的 lambda_ 最优值。

## 8. 结论

1. **AdaptiveFlippedLasso 在 Exp1 场景下表现优异**，最佳 F1 达到 0.9486，TPR=100%，FDR≈10%
2. **lambda_ 是最关键参数**，建议在 [0.5, 1.0] 范围内进一步细调
3. **gamma=0.5（弱分化）配合 lambda_=1.0 是最优组合**，实现了良好的稀疏性-准确性平衡
4. **lambda_ridge 影响可忽略**，第一阶段 Ridge 强度不敏感
5. **gamma=2.0 不推荐**，分化过强导致算法失效
